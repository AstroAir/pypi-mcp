## Open Source Dependencies

The PyPI MCP Server is built on the shoulders of giants. We acknowledge and thank the maintainers and contributors of these essential open source projects:

### Core Framework

- **[FastMCP](https://github.com/jlowin/fastmcp)** by Marvin Team
  - Modern MCP framework that powers our server
  - Provides async/await support and clean API design
  - Licensed under MIT License

### HTTP and Networking

- **[httpx](https://github.com/encode/httpx)** by Tom Christie and contributors
  - Async HTTP client for Python
  - Excellent performance and modern async/await support
  - Licensed under BSD-3-Clause License

### Data Validation and Serialization

- **[Pydantic](https://github.com/pydantic/pydantic)** by Samuel Colvin and contributors
  - Data validation using Python type annotations
  - Ensures type safety and data integrity
  - Licensed under MIT License

### Package Management

- **[packaging](https://github.com/pypa/packaging)** by Python Packaging Authority
  - Core utilities for Python packages
  - Version parsing and requirement handling
  - Licensed under Apache-2.0 OR BSD-2-Clause

### Caching

- **[cachetools](https://github.com/tkem/cachetools)** by Thomas Kemmer
  - Extensible memoizing collections and decorators

  - Licensed under MIT License

## How to Be Acknowledged

We welcome contributions of all kinds and recognize them in various ways:

### Code Contributions

- **Pull Requests**: All merged pull requests are automatically credited
- **GitHub Contributors**: Listed on the repository contributors page
- **Release Notes**: Significant contributions mentioned in release notes

### Non-Code Contributions

- **Documentation**: Improvements to documentation and examples
- **Bug Reports**: High-quality bug reports that help improve the project
- **Feature Requests**: Thoughtful feature suggestions that shape the roadmap
- **Community Support**: Helping other users in discussions and issues

### Recognition Levels

- **Contributors**: Anyone who has made a contribution to the project
- **Regular Contributors**: Contributors with multiple significant contributions
- **Core Contributors**: Long-term contributors with significant impact
- **Maintainers**: Contributors with commit access and project oversight

## Contributing Recognition

To ensure proper recognition:

1. **Use your real name** or preferred professional name in commits
2. **Include your GitHub username** for proper attribution
3. **Update documentation** if you add significant features
4. **Participate in discussions** to help shape the project direction

## Contact for Acknowledgments

If you believe your contribution should be acknowledged or if there are any errors in the acknowledgments:

- **GitHub Issues**: Create an issue with the "acknowledgment" label
- **Pull Request**: Submit a PR to update this file
- **Direct Contact**: Reach out to the maintainers
